# DDoS 攻擊控制台使用說明

## 📋 功能特點

✅ **網頁圖形化介面**
- 直觀的攻擊類型選擇按鈕
- 實時進度條顯示
- 線程數和持續時間滑桿調整
- 實時統計數據更新
- 攻擊日誌記錄

✅ **支持的攻擊類型**
1. **SYN Flood** - 半開連接攻擊
2. **HTTP GET Flood** - GET 請求洪水
3. **HTTP POST Flood** - POST 請求洪水
4. **Slowloris** - 慢速連接耗盡
5. **UDP Flood** - UDP 封包洪水
6. **組合攻擊** - 多種攻擊同時進行

## 🚀 啟動步驟

### 1. 安裝依賴

```powershell
pip install flask flask-cors requests
```

### 2. 啟動後端服務器

```powershell
cd topic\DDos\dos
python attack_server.py
```

服務器會啟動在 `http://localhost:5000`

### 3. 打開網頁控制台

在瀏覽器中訪問: `http://localhost:5000`

### 4. 使用控制台

1. **配置目標**
   - 輸入目標 IP（測試用 127.0.0.1）
   - 設置 TCP 端口（默認 8000）
   - 設置 UDP 端口（默認 9001）

2. **調整參數**
   - 使用滑桿調整線程數（1-200）
   - 設置攻擊持續時間（5-300 秒）

3. **選擇攻擊類型**
   - 點擊任意攻擊按鈕選擇
   - 按鈕會高亮顯示當前選擇

4. **開始攻擊**
   - 點擊 "▶️ 開始攻擊" 按鈕
   - 觀察進度條和統計數據
   - 查看實時日誌

5. **停止/重置**
   - 點擊 "⏹️ 停止攻擊" 提前結束
   - 點擊 "🔄 重置" 清空所有數據

## 📊 統計數據說明

- **發送封包**: UDP/ICMP 封包總數
- **建立連接**: TCP 連接總數
- **HTTP 請求**: HTTP GET/POST 請求數
- **運行時間**: 攻擊已運行時長

## ⚠️ 重要警告

1. **僅用於測試自己的伺服器**
2. **未經授權攻擊他人伺服器屬於違法行為**
3. **建議在本地環境測試 (127.0.0.1)**
4. **確保目標伺服器是您擁有和控制的**

## 🔧 測試流程

### 完整測試步驟

1. **啟動監控伺服器** (終端 1)
   ```powershell
   cd topic\DDos\server
   python muti_server.py
   ```
   打開 http://localhost:8888 查看監控面板

2. **啟動攻擊控制台** (終端 2)
   ```powershell
   cd topic\DDos\dos
   python attack_server.py
   ```
   打開 http://localhost:5000 操作攻擊

3. **執行攻擊測試**
   - 在控制台選擇攻擊類型
   - 調整線程數（建議 50-100）
   - 設置持續時間（建議 30 秒）
   - 點擊開始攻擊

4. **觀察效果**
   - 監控面板會顯示攻擊數據
   - 攻擊控制台顯示發送統計
   - 比對兩邊的數據變化

## 💡 推薦配置

### 高效測試組合
- **攻擊類型**: 組合攻擊
- **線程數**: 50-100
- **持續時間**: 30 秒
- **效果**: 多種攻擊同時進行，效果最明顯

### 單一攻擊測試
- **HTTP GET/POST**: 50 線程，30 秒
- **SYN Flood**: 100 線程，30 秒
- **UDP Flood**: 50 線程，30 秒

## 🐛 故障排除

### 控制台無法連接
- 確認 `attack_server.py` 正在運行
- 檢查端口 5000 是否被占用
- 嘗試重啟後端服務器

### 統計數據不更新
- 刷新瀏覽器頁面
- 檢查瀏覽器控制台是否有錯誤
- 確認後端 API 正常響應

### 攻擊效果不明顯
- 增加線程數
- 確認目標伺服器正在運行
- 檢查防火牆設置
