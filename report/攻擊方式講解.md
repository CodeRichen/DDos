## 📚 真實世界的伺服器配置

### **典型的網站伺服器**
```
公司官網 (example.com)
├─ 80  (HTTP)   → 重定向到 HTTPS
├─ 443 (HTTPS)  → 主要服務
└─ 其他端口關閉

有效攻擊: HTTP/HTTPS Flood, Slowloris
無效攻擊: UDP Flood, DNS 攻擊
```

### **遊戲伺服器**
```
遊戲伺服器 (game.example.com)
├─ 7777 (遊戲協議) → UDP
├─ 80   (網頁面板)  → TCP
└─ 27015 (查詢)    → UDP

有效攻擊: UDP Flood (遊戲端口)
         HTTP Flood (網頁端口)
```

### **企業伺服器**
```
企業內網伺服器
├─ 22   (SSH)
├─ 80   (HTTP)
├─ 443  (HTTPS)
├─ 3306 (MySQL)
└─ 3389 (RDP)

有效攻擊: 幾乎所有 TCP 攻擊
特別注意: 資料庫端口最脆弱
```

## 🔍 如何判斷攻擊方式

### **簡單判斷流程**
```
1. 先掃描端口
   ↓
2. 看開放什麼服務
   ↓
3. 選擇對應攻擊

端口 80/443    → HTTP Flood
端口 53        → DNS Flood + UDP Flood
端口 22        → SYN Flood + SSH 耗盡
任何 TCP 端口  → SYN Flood
任何 UDP 端口  → UDP Flood
```
